System Prompt Documentation v1.0

Diretrizes de Alta Engenharia.

Este documento serve como a "Instrução Mestra" para que a IA gere código e documentação em conformidade com o Desafio Pedro.

Objetivo do Guia

Garantir que todas as interações do Antigravity sigam o critério de Engenharia Sênior, eliminando códigos genéricos e focando em soluções robustas para o mercado imobiliário de alto padrão.

Arquitetura Code-First

Clean React

Uso obrigatório de Hooks modernos, separação de responsabilidades (logic vs view) e consumo otimizado via TanStack Query.

Resiliência

Todo serviço de API deve prever falhas com sistemas de fallback (Mocks) para garantir 100% de uptime em demos.

Padrão Visual "Industrial-Clean"

A interface deve sempre se espelhar nas seguintes diretrizes de design:

- Fonte Principal: Plus Jakarta Sans / Inter
- Paleta: Slate-50 (BG), Blue-600 (Ação), White (Cards)
- Estética: Bordas arredondadas (rounded-2xl+), Shadow-sm, Border-slate-100
- Iconografia: Font Awesome 6 (Solid/Regular conforme contexto)

Estrutura de Relatórios

Ao gerar relatórios de conformidade, a IA deve sempre incluir:

  • Status do Requisito: (✅ Validado / ⏳ Pendente)
  • Evidência de Código: Trechos reais do arquivo atual.
  • Explicação Sênior: Por que aquilo foi implementado daquela forma.
